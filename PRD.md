# Korean BBS PRD

## 1. 제품 개요

Korean BBS는 Laravel 애플리케이션에 빠르게 붙일 수 있는 한국형 게시판 패키지다. 그누보드5, KBoard에 익숙한 국내 서비스 운영자가 기대하는 다중 게시판, 공지, 비밀글, 비회원 글쓰기, 댓글, 첨부파일, 갤러리 스킨, 관리자 화면을 Laravel·Livewire 기반으로 제공한다.

이 제품은 독립형 서비스가 아니라 Composer로 설치되는 Laravel 라이브러리이며, 호스트 애플리케이션의 인증, 레이아웃, 파일 스토리지, 라우팅 설정과 자연스럽게 연동되는 것을 목표로 한다.

## 2. 문제 정의

Laravel 프로젝트에서 한국형 게시판을 매번 새로 구현하면 다음 비용이 반복된다.

- 게시판별 설정, 권한, 스킨, 페이징, 검색 구현 비용
- 비회원 작성자 정보와 비밀번호 기반 비밀글 처리 비용
- 첨부파일, 이미지 썸네일, 갤러리형 목록 구현 비용
- 관리자용 게시판·게시글·댓글 관리 화면 구현 비용
- 기존 Laravel 앱의 레이아웃과 인증 체계에 맞추는 통합 비용

Korean BBS는 이 반복 비용을 줄이고, 패키지 설치 후 설정과 마이그레이션만으로 운영 가능한 게시판 경험을 제공해야 한다.

## 3. 목표

- Laravel 11 이상 프로젝트에서 Composer 설치, 설정 publish, migrate 후 바로 사용 가능한 게시판 패키지를 제공한다.
- 일반 목록형 게시판과 갤러리형 게시판을 모두 지원한다.
- 회원/비회원 혼합 사용 환경을 지원한다.
- 운영자가 코드 수정 없이 관리자 화면에서 게시판을 생성, 수정, 삭제하고 게시글·댓글을 관리할 수 있게 한다.
- 호스트 애플리케이션이 Blade 레이아웃, URL prefix, 업로드 설정, 스킨을 커스터마이징할 수 있게 한다.

## 4. 비목표

- 완전한 CMS 또는 커뮤니티 플랫폼 전체 기능을 제공하지 않는다.
- 호스트 애플리케이션의 사용자 인증 시스템을 대체하지 않는다.
- WYSIWYG 에디터, 포인트/레벨 시스템, 쪽지, 신고, 차단, 소셜 로그인은 초기 핵심 범위에 포함하지 않는다.
- 대규모 CDN 이미지 변환, 바이러스 검사, 전문 검색 엔진 연동은 기본 패키지 책임 범위에 포함하지 않는다.

## 5. 대상 사용자

### 5.1 Laravel 개발자

- 기존 Laravel 프로젝트에 게시판 기능을 빠르게 추가하려는 개발자
- 패키지의 기본 뷰를 publish 후 프로젝트 디자인에 맞게 수정하려는 개발자
- 게시판 스킨을 추가하거나 기본 설정을 확장하려는 개발자

### 5.2 사이트 운영자

- 관리자 화면에서 게시판을 만들고 관리하려는 운영자
- 공지글, 일반글, 댓글을 모니터링하고 삭제해야 하는 운영자
- 목록형/갤러리형 게시판을 용도별로 운영하려는 운영자

### 5.3 게시판 이용자

- 회원 또는 비회원으로 글을 작성하려는 사용자
- 댓글과 답글을 작성하려는 사용자
- 첨부파일을 내려받거나 이미지가 포함된 게시글을 열람하려는 사용자

## 6. 핵심 사용자 시나리오

### 6.1 패키지 설치

1. 개발자는 Composer로 `ssh521/korean-bbs`를 설치한다.
2. 개발자는 설정 파일을 publish한다.
3. 개발자는 migration을 실행하고 storage link를 생성한다.
4. 개발자는 `/bbs`와 `/bbs-admin` 경로에서 게시판과 관리자 화면에 접근한다.

### 6.2 게시판 생성

1. 운영자는 `/bbs-admin/login`에서 관리자 계정으로 로그인한다.
2. 운영자는 게시판 관리 화면에서 새 게시판을 생성한다.
3. 운영자는 게시판 이름, slug, 설명, 스킨, 페이지당 글 수, 댓글/추천/첨부/비밀글 사용 여부를 설정한다.
4. 이용자는 `/bbs/{slug}`에서 생성된 게시판을 볼 수 있다.

### 6.3 게시글 작성과 열람

1. 이용자는 게시판 목록에서 글쓰기 화면으로 이동한다.
2. 회원은 로그인 계정으로, 비회원은 이름과 비밀번호를 입력해 글을 작성한다.
3. 첨부 기능이 활성화된 게시판에서는 허용 확장자와 크기 제한 내에서 파일을 업로드한다.
4. 이용자는 상세 화면에서 본문, 첨부파일, 댓글, 추천/비추천을 확인한다.

### 6.4 비밀글 열람

1. 작성자는 비밀글 허용 게시판에서 비밀글을 작성한다.
2. 비회원 작성 글은 작성 시 설정한 비밀번호로 보호된다.
3. 상세 화면 접근 시 올바른 비밀번호를 입력한 사용자만 본문을 열람한다.

### 6.5 관리자 콘텐츠 관리

1. 운영자는 관리자 대시보드에서 게시판, 게시글, 댓글 수와 최근 글을 확인한다.
2. 운영자는 게시글 목록에서 게시판별 필터와 검색을 사용한다.
3. 운영자는 개별 또는 선택된 게시글을 삭제한다.
4. 운영자는 댓글 검색 후 부적절한 댓글을 삭제한다.

## 7. 기능 요구사항

### 7.1 패키지 설치와 설정

- Composer 패키지로 설치할 수 있어야 한다.
- Laravel service provider를 통해 migration, view, route, Livewire component를 자동 등록해야 한다.
- 설정 파일 publish를 지원해야 한다.
- 공개 게시판 prefix와 관리자 prefix를 설정으로 변경할 수 있어야 한다.
- 공개 페이지 Blade layout을 설정으로 교체할 수 있어야 한다.
- 공개 페이지 layout은 현재 페이지 제목, Breadcrumb, 게시판, 게시글 정보를 props로 받을 수 있어야 한다.
- 파일 업로드 disk, path, 최대 크기, 허용 확장자, 이미지 확장자, 썸네일 크기를 설정할 수 있어야 한다.

### 7.2 게시판 그룹

- 게시판은 선택적으로 그룹에 속할 수 있어야 한다.
- 그룹은 이름, slug, 정렬 순서를 가져야 한다.
- 공개 메인 화면은 활성 게시판을 그룹과 함께 보여줄 수 있어야 한다.

### 7.3 게시판 관리

- 관리자는 게시판을 생성, 수정, 삭제할 수 있어야 한다.
- 게시판은 이름, slug, 설명, 스킨, 공개 화면 width, 그룹, 권한 레벨, 페이지당 글 수, 기능 사용 여부, 활성 상태, 정렬 순서를 가져야 한다.
- slug는 전체 게시판에서 유일해야 한다.
- 허용되지 않은 스킨 키는 저장할 수 없어야 한다.
- 공개 화면 width는 TailwindCSS 클래스 또는 `100%`, `600px` 같은 CSS width 값으로 지정할 수 있어야 한다.
- 비활성 게시판은 공개 화면에서 접근할 수 없어야 한다.

### 7.4 게시글 목록

- 게시글 목록은 공지글과 일반글을 구분해 표시해야 한다.
- 공지글은 일반 페이징 목록과 별도로 노출되어야 한다.
- 일반 목록은 최신순으로 정렬되어야 한다.
- 제목, 내용, 작성자 기준 검색을 지원해야 한다.
- 일반 스킨은 게시판별 페이지당 글 수를 사용해야 한다.
- 갤러리 스킨은 갤러리 전용 기본 페이지당 개수를 사용할 수 있어야 한다.
- 댓글 수, 조회 수, 추천 수, 작성일 등 게시판 운영에 필요한 주요 메타데이터를 노출할 수 있어야 한다.

### 7.5 게시글 작성과 수정

- 회원은 로그인 사용자 ID로 게시글을 작성할 수 있어야 한다.
- 비회원은 작성자 이름과 비밀번호를 입력해 게시글을 작성할 수 있어야 한다.
- 제목은 필수이며 최대 200자여야 한다.
- 본문은 필수여야 한다.
- 관리자는 공지글을 작성할 수 있어야 한다.
- 게시판에서 비밀글을 허용한 경우 작성자는 비밀글 여부를 선택할 수 있어야 한다.
- 게시글 수정 시 기존 제목, 본문, 공지, 비밀글 상태를 불러와야 한다.

### 7.6 비밀글

- 비밀글이 아닌 게시글은 상세 접근 시 조회 수가 증가해야 한다.
- 비밀글은 잠금 상태로 시작해야 한다.
- 올바른 작성자 비밀번호를 입력하면 본문을 열람할 수 있어야 한다.
- 비밀글 잠금 해제 후 조회 수가 증가해야 한다.

### 7.7 댓글과 답글

- 게시판에서 댓글 기능이 활성화된 경우 댓글을 작성할 수 있어야 한다.
- 회원은 로그인 사용자 ID로 댓글을 작성할 수 있어야 한다.
- 비회원은 작성자 이름과 비밀번호를 입력해 댓글을 작성할 수 있어야 한다.
- 댓글 본문은 필수이며 최대 2,000자여야 한다.
- 댓글은 parent_id를 통해 답글을 지원해야 한다.
- 댓글과 답글은 작성일 오름차순으로 표시되어야 한다.
- 댓글은 soft delete를 지원해야 한다.

### 7.8 추천과 비추천

- 게시글은 추천과 비추천을 받을 수 있어야 한다.
- 로그인 사용자는 사용자 ID 기준으로 중복 반응이 제한되어야 한다.
- 비회원은 IP 주소 기준으로 중복 반응이 제한되어야 한다.
- 사용자가 기존 반응과 다른 타입을 선택하면 기존 카운트를 차감하고 새 카운트를 증가시켜야 한다.
- 댓글도 추천/비추천 데이터 구조를 지원해야 한다.

### 7.9 파일 첨부와 다운로드

- 게시판에서 첨부 기능이 활성화된 경우 게시글에 파일을 첨부할 수 있어야 한다.
- 파일 업로드는 설정된 최대 크기와 허용 확장자를 따라야 한다.
- 파일은 설정된 disk와 path 하위에 게시판 ID 기준으로 저장되어야 한다.
- 원본 파일명, 저장 파일명, 경로, MIME type, 크기, 이미지 여부, 다운로드 수를 저장해야 한다.
- 이미지 파일은 미리보기와 갤러리 썸네일로 사용할 수 있어야 한다.
- 첫 번째 이미지 파일은 게시글 썸네일로 설정될 수 있어야 한다.
- 다운로드 시 다운로드 수가 증가해야 한다.

### 7.10 스킨

- 기본 스킨으로 `list`, `gallery`, `custom`을 제공해야 한다.
- 스킨별로 목록, 상세, 작성 화면 Blade를 분리해야 한다.
- Breadcrumb은 개별 스킨이 아니라 layout에서 공통 처리할 수 있어야 한다.
- 설정에서 허용 스킨 목록을 확장할 수 있어야 한다.
- 존재하지 않거나 허용되지 않은 스킨은 안전하게 처리해야 한다.
- 개발자는 view publish 후 프로젝트 디자인에 맞게 스킨을 수정할 수 있어야 한다.

### 7.11 관리자 인증

- 관리자 로그인은 패키지 설정의 관리자 ID, 비밀번호, 이름을 사용해야 한다.
- 관리자 인증 상태는 세션으로 유지되어야 한다.
- 관리자 전용 route는 인증 middleware로 보호되어야 한다.
- 관리자는 로그아웃할 수 있어야 한다.

### 7.12 관리자 대시보드

- 관리자는 게시판 수, 게시글 수, 댓글 수를 확인할 수 있어야 한다.
- 관리자는 최근 게시글 목록을 확인할 수 있어야 한다.

### 7.13 관리자 게시글 관리

- 관리자는 게시글을 최신순으로 조회할 수 있어야 한다.
- 관리자는 제목으로 게시글을 검색할 수 있어야 한다.
- 관리자는 게시판별로 게시글을 필터링할 수 있어야 한다.
- 관리자는 게시글을 개별 삭제할 수 있어야 한다.
- 관리자는 현재 페이지의 게시글을 선택해 일괄 삭제할 수 있어야 한다.

### 7.14 관리자 댓글 관리

- 관리자는 댓글을 최신순으로 조회할 수 있어야 한다.
- 관리자는 댓글 본문으로 검색할 수 있어야 한다.
- 관리자는 댓글이 속한 게시글과 게시판 정보를 확인할 수 있어야 한다.
- 관리자는 댓글을 삭제할 수 있어야 한다.

## 8. 권한 정책

현재 권한 레벨은 다음 의미를 기본값으로 한다.

- `0`: 비회원 허용
- `1`: 로그인 회원 필요

게시판은 `write_level`, `comment_level`, `file_level`을 저장해야 하며, 각 사용자 액션은 해당 설정을 기준으로 접근 가능 여부를 판단해야 한다. 향후 호스트 앱의 회원 등급 시스템과 연동할 수 있도록 숫자 기반 레벨을 유지한다.

## 10. URL 요구사항

기본 공개 URL은 다음을 제공해야 한다.

- `GET /bbs`: 게시판 목록
- `GET /bbs/{slug}`: 게시글 목록
- `GET /bbs/{slug}/create`: 게시글 작성
- `GET /bbs/{slug}/{post}`: 게시글 상세
- `GET /bbs/{slug}/{post}/edit`: 게시글 수정
- `GET /bbs/file/{file}/download`: 파일 다운로드

기본 관리자 URL은 다음을 제공해야 한다.

- `GET /bbs-admin/login`: 관리자 로그인 화면
- `POST /bbs-admin/login`: 관리자 로그인 처리
- `POST /bbs-admin/logout`: 관리자 로그아웃
- `GET /bbs-admin`: 관리자 대시보드
- `GET /bbs-admin/boards`: 게시판 관리
- `GET /bbs-admin/boards/create`: 게시판 생성
- `GET /bbs-admin/boards/{board}/edit`: 게시판 수정
- `GET /bbs-admin/posts`: 게시글 관리
- `GET /bbs-admin/comments`: 댓글 관리

## 11. 비기능 요구사항

### 11.1 호환성

- PHP `^8.2`를 지원해야 한다.
- Laravel `^11.0`, `^12.0`, `^13.0`을 지원해야 한다.
- Livewire `^3.0`을 지원해야 한다.

### 11.2 보안

- 비회원 게시글과 댓글 비밀번호는 해시로 저장해야 한다.
- 관리자 비밀번호는 운영 환경에서 `.env`로 설정해야 한다.
- 파일 업로드는 확장자와 크기 제한을 검증해야 한다.
- 관리자 route는 인증 middleware로 보호해야 한다.
- 게시글 삭제는 작성자 또는 관리자만 가능해야 한다.
- 비밀글 본문은 잠금 해제 전 노출되지 않아야 한다.

### 11.3 성능

- 게시글 목록은 pagination을 사용해야 한다.
- 관리자 목록도 pagination을 사용해야 한다.
- 게시글 목록 조회 시 댓글 수는 count query로 제공해야 한다.
- 파일과 이미지 리소스는 Laravel storage disk를 통해 제공해야 한다.

### 11.4 확장성

- Blade view publish를 통해 UI를 커스터마이징할 수 있어야 한다.
- 스킨 resolver를 통해 스킨별 list/show/form view를 분리해야 한다.
- 설정 파일로 URL prefix, layout, upload, defaults, skins를 조정할 수 있어야 한다.

### 11.5 접근성과 사용성

- 기본 화면은 TailwindCSS 기반의 반응형 UI를 제공해야 한다.
- 검색, 필터, 작성, 삭제 같은 주요 동작은 명확한 상태와 피드백을 제공해야 한다.
- 관리자 삭제 작업은 확인 절차를 제공해야 한다.

## 13. 성공 지표

- 새 Laravel 프로젝트에서 설치부터 첫 게시판 생성까지 10분 이내 완료
- 기본 설정만으로 공개 게시판과 관리자 화면이 정상 동작
- view publish 후 프로젝트 레이아웃에 무리 없이 통합
- 게시글 목록, 상세, 작성, 댓글, 첨부, 관리자 삭제 흐름에서 치명적 오류 0건
- 패키지 업데이트 시 기존 게시판 데이터 migration 호환성 유지

## 14. 주요 리스크와 대응

- 호스트 앱 인증 구조 차이: Laravel 기본 auth model 설정을 따르고, 관리자 인증은 패키지 세션 인증으로 분리한다.
- 업로드 보안: 확장자, 크기 제한을 기본 제공하고 운영자가 config로 조정하게 한다.
- 스킨 확장 복잡도: SkinResolver와 view naming convention을 유지해 새 스킨 추가 비용을 낮춘다.
- Laravel 버전별 차이: 지원 버전별 Testbench 테스트를 추가해 service provider, route, migration 동작을 검증한다.
- 한국형 게시판 기대 기능의 범위 확장: MVP와 이후 개선 후보를 분리해 핵심 설치형 게시판 경험을 먼저 안정화한다.

## 15. 오픈 이슈

- 게시판 권한 레벨을 실제 사용자 등급과 어떻게 매핑할지 정의가 필요하다.
- 비회원 작성글 수정/삭제 시 비밀번호 검증 UX가 필요하다.
- 게시판 그룹 생성/수정/삭제 관리자 화면이 필요하다.
- 추천/비추천 중복 제한에서 비회원 IP 기준 정책의 운영상 한계를 문서화해야 한다.
- 갤러리 썸네일 리사이징을 실제 이미지 처리로 구현할지, 원본 이미지를 그대로 사용할지 결정해야 한다.
