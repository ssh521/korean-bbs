# Korean BBS Design Principles

## 1) 목적
- 이 문서는 `korean-bbs`의 UI/UX 일관성을 위한 최상위 원칙이다.
- 관리자는 "작업 효율"을, 커뮤니티/스킨은 "표현 유연성"을 우선한다.

## 2) 범위
- 적용 대상:
  - 관리자 화면 전체
  - 기본 커뮤니티 화면
  - 스킨 개발 가이드
- 비적용 대상:
  - 외부 서비스 대시보드
  - 운영 환경별 브랜딩 문구

## 3) 핵심 결정사항
1. 관리자 페이지는 스킨을 적용하지 않는다.
2. 관리자 페이지는 색상 토큰만 변경 가능하다.
3. 브랜드 컬러(Primary/Success/Warning/Danger)는 고정 팔레트를 사용한다.
4. Tailwind는 CDN 기반에서 빌드 기반(CLI/Vite)으로 전환한다.
5. 관리자 화면은 모바일 완전 대응을 목표로 한다.

## 4) 디자인 원칙
1. Data-first: 먼저 데이터 파악, 그 다음 액션.
2. Predictable: 페이지마다 버튼/입력/테이블 동작이 같아야 한다.
3. Mobile-first: 모바일에서 깨지지 않고, 데스크톱에서 확장된다.
4. Accessible-by-default: 대비, 포커스, 라벨을 기본 제공한다.
5. Reusable: 공통 UI를 컴포넌트로 묶어 중복을 줄인다.

## 5) 관리자 vs 스킨 정책
| 구분 | 관리자(Admin) | 커뮤니티 스킨(Skins) |
|---|---|---|
| 레이아웃 | 고정 | 자유 |
| 컴포넌트 | 공통 컴포넌트 강제 | 선택 사용 가능 |
| 색상 | 토큰 변경만 허용 | 토큰 기반 확장 허용 |
| 데이터 슬롯 | 고정 | 계약 내에서 확장 |
| 목표 | 효율/정확성 | 다양성/브랜딩 |

## 6) 금지사항
1. 관리자 페이지별 임의 버튼 스타일 분기
2. 관리자 페이지에 스킨별 템플릿 주입
3. 모바일에서 강제 가로 스크롤 전제 UI
4. 상태 색상 임의 변경

## 7) 운영 원칙
1. UI 변경 PR에는 `docs/design/05-review-checklist.md` 체크를 포함한다.
2. 관리자 UI 변경 시 공통 컴포넌트 우선 수정 후 개별 페이지 적용.
3. 스킨 관련 변경은 `docs/design/03-skin-contract-v1.md`를 기준으로 검토한다.
