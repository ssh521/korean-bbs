# Korean BBS Design Review Checklist

## 공통
- [ ] 화면 여백/타이포/버튼 스타일이 페이지 간 일관적인가?
- [ ] 반복 UI가 공통 컴포넌트를 사용하도록 구현되었는가?
- [ ] 비정상 상태(Empty/Loading/Error)가 누락되지 않았는가?

## 관리자 페이지
- [ ] 관리자 화면에 스킨 주입 로직이 없는가?
- [ ] 관리자 화면 색상 변경이 토큰 기반으로만 이루어졌는가?
- [ ] List Page에 검색/필터/Row Action/페이지네이션이 제공되는가?
- [ ] Bulk Action이 선택 상태와 연동되는가?
- [ ] 모바일에서 카드형 리스트로 자연스럽게 전환되는가?
- [ ] Danger 액션에 확인 절차가 포함되는가?

## 스킨(v1)
- [ ] `list/show/form` 3개 뷰가 모두 존재하는가?
- [ ] Empty 상태에서 사용자 안내 문구가 있는가?
- [ ] 공지/비밀글/권한 기반 렌더링이 깨지지 않는가?
- [ ] 페이지네이션 연동이 정상 동작하는가?

## 접근성(현재 권장)
- [ ] 입력 필드에 label 또는 동등한 접근성 라벨이 있는가?
- [ ] 키보드 포커스가 시각적으로 확인 가능한가?
- [ ] 색상만으로 상태를 전달하지 않는가?

## 릴리즈 전 최종 확인
- [ ] `docs/design/01-principles.md`와 충돌되는 구현이 없는가?
- [ ] `docs/design/02-admin-harness.md`의 구조를 어기지 않았는가?
- [ ] `docs/design/03-skin-contract-v1.md` 계약을 어기지 않았는가?
- [ ] `docs/design/04-color-tokens.md` 고정 팔레트를 유지했는가?
