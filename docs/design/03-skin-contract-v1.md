# Korean BBS Skin Contract v1

## 1) 이 문서의 의미
- 스킨 계약(Contract)은 "코어 업데이트 후에도 스킨이 깨지지 않도록 지키는 약속"이다.
- 현재 프로젝트는 `v1`만 운영한다.

## 2) 적용 범위
- 대상: 게시판 스킨 뷰(`list/show/form`)
- 비대상: 관리자 페이지(관리자는 스킨 비적용)

## 3) 필수 파일 규칙
스킨은 아래 3개 뷰를 반드시 제공해야 한다.
1. `list.blade.php`
2. `show.blade.php`
3. `form.blade.php`

`SkinResolver`는 위 타입만 지원한다.

## 4) 필수 데이터 계약
스킨은 아래 공통 데이터를 정상적으로 렌더링해야 한다.
1. 게시판: `board.name`, `board.slug`
2. 게시글: `post.id`, `post.title`, `post.author_name`, `post.created_at`
3. 목록: `posts`(페이지네이션 컬렉션)
4. 권한: 읽기/쓰기 가능 여부 기반 버튼/링크 처리

## 5) 필수 상태 계약
1. Empty 상태: 데이터가 없을 때 안내 문구 제공
2. Pagination 상태: 페이지네이션 컴포넌트와 호환
3. 비밀글/공지글 상태: 텍스트로 구분 표시

## 6) 접근성 최소 계약
1. 이미지에는 대체 텍스트를 제공한다.
2. 링크/버튼은 키보드 포커스 가능해야 한다.
3. 색상만으로 의미를 전달하지 않는다.

## 7) 금지사항
1. 코어 내부 라우트/메서드명을 하드코딩으로 덮어쓰기
2. 관리자 화면 스타일 오염
3. 필수 뷰 타입 누락

## 8) 하위 호환 정책(v1)
1. 코어는 v1 필수 계약을 임의로 제거하지 않는다.
2. 계약을 바꿔야 할 정도의 변경이 생기면 `v2` 문서를 추가한다.
3. `v2` 도입 시에도 일정 기간 `v1` 지원을 유지한다.

## 9) 스킨 개발 체크
1. `list/show/form` 3종 제공 여부
2. Empty/공지/비밀글 처리 여부
3. 모바일 레이아웃 깨짐 여부
4. 페이지네이션 연동 여부
